# 2027 考研英语词书微信小程序设计规格

## 1. 项目目标

开发一款供本人及少数朋友使用的考研英语词书微信小程序。产品以纸质词书的连续阅读体验为核心，每个单词显示为一个完整模块，方便用户阅读、检索、分类和手动复习。

首版采用原生微信小程序与微信云开发，兼容 iOS 和 Android 微信。词库及公共音频存放于云端，按词库包下载；个人学习记录只保存在各自设备本地，不提供账号、云同步或排行榜。

## 2. 用户与发布方式

- 目标用户：正在准备 2027 年考研英语的本人及少数朋友。
- 发布方式：微信小程序体验版，不面向公众正式发布。
- 设备范围：主流 iPhone 和 Android 手机中的微信客户端。
- 数据隔离：每台设备独立保存学习记录，用户之间互不影响。

## 3. 首版范围

### 3.1 包含功能

1. 词库中心
   - 展示可下载词库及版本信息。
   - 支持单独下载、更新和删除词库包。
   - 首批规划“大纲词汇”“核心词汇”“真题高频词”等分类。
   - 已下载词库支持离线阅读。

2. 词书阅读
   - 采用类似纸质词书的上下连续阅读方式。
   - 每个单词显示为一个独立模块，不使用翻面卡片。
   - 支持搜索、字母索引、词库筛选和熟悉度筛选。
   - 自动记录并恢复各词库的上次阅读位置。
   - 支持调整正文字号。

3. 熟悉度管理
   - 每个单词可标记为“熟悉”“待巩固”或“陌生”。
   - 用户可按熟悉度和词库范围进入连续复习。
   - 复习由用户手动发起，不提供自动复习日程和提醒。

4. 发音
   - 支持英音和美音播放与切换。
   - 支持音频缓存，缓存成功后可离线播放。
   - 音频失败不阻塞词条阅读。

5. 单词测试
   - 看英文选择中文释义。
   - 看中文释义选择英文单词。
   - 测试结束显示题数、正确数、正确率和错词。
   - 用户可将错词标记为“待巩固”或“陌生”。

### 3.2 首版不包含

- 翻面卡片模式。
- 微信登录和用户账号体系。
- 学习记录云同步和换机恢复。
- 好友排行或社交功能。
- 自动间隔复习、学习提醒或每日任务。
- 拼写测试、听音辨词。
- 用户自行编辑或上传词条。
- 面向公众的正式发布、付费或商业化功能。

## 4. 页面与主要流程

### 4.1 首页

显示：

- 已下载词库及继续阅读入口。
- 已阅读或已标记的单词数量。
- “熟悉”“待巩固”“陌生”数量统计。
- 词库中心、分类复习和单词测试入口。

### 4.2 词库中心

每个词库展示：

- 名称、简介和词条数量。
- 内容版本和更新时间。
- 下载状态与预计占用空间。
- 下载、更新、删除操作。

下载采用临时文件加校验的方式：下载和解析全部成功后才替换旧版本。中断或校验失败时继续保留旧版本。

### 4.3 词书阅读页

默认按词库预设顺序连续展示词条。页面提供：

- 返回顶部、继续阅读和当前位置提示。
- 单词搜索。
- 字母索引。
- 词库与熟悉度筛选。
- 字号调整。
- 英音、美音播放。
- 三档熟悉度标记。

为了保证长词库滚动性能，页面只渲染当前位置附近的有限词条，不一次性创建全部词条节点。

### 4.4 分类复习页

用户先选择：

- 一个或多个已下载词库。
- “熟悉”“待巩固”“陌生”中的一个或多个状态。

之后进入与词书阅读页一致的连续阅读界面。筛选结果不改变原词库顺序。

### 4.5 单词测试页

用户选择测试类型、词库范围、熟悉度范围和题量。每题提供一个正确答案与若干干扰项。测试结束后显示结果并提供错词列表。

干扰项优先从相同词性或相近难度的词条中生成；候选不足时再从当前词库随机补足。测试记录首版只保留最近一次结果，不建设长期统计系统。

## 5. 词条模块设计

视觉上参考传统考研词书的信息层级和紧凑阅读感，但不复制任何商业词书的具体版式、插图、文案或受保护内容。

每个词条模块按以下顺序展示：

1. 序号、英文单词、词频或重要度标签、熟悉度状态。
2. 英式音标、美式音标及对应发音按钮。
3. 词性和中文释义。
4. 常用搭配。
5. 词根词缀与记忆提示。
6. 近义词、反义词和易混词。
7. 真题短句、中文翻译、年份、题型及来源。
8. “熟悉”“待巩固”“陌生”操作按钮。

空字段对应的栏目直接隐藏，避免出现空白标题。

视觉原则：

- 米白或浅暖色页面背景，深色正文。
- 使用一种主要强调色标记重点释义和交互状态。
- 通过留白、浅色底块或细分隔线区分词条。
- 正文和按钮满足小屏可读性与触控尺寸要求。
- 不依赖仅在 iOS 或 Android 上成立的字体、样式或交互。

## 6. 词库内容与版权边界

### 6.1 内容阶段

1. 试运行词库：先整理约 500–1000 个核心或高频词，用于验证产品体验和数据流程。
2. 大纲词库：依据可核验的最新官方考研英语大纲补齐词汇。
3. 深度内容：逐步补充真题短句、搭配、词根词缀、近反义词和易混词。

如果 2027 年考研英语正式大纲尚未发布，则词库明确标注所依据的最新官方大纲年份；新大纲发布后再新增版本并提供更新。

### 6.2 来源要求

- 每条核心内容记录来源类型和必要的来源标识。
- 优先使用官方材料、开放许可资源、公版词典或自行整理内容。
- 不直接抓取或复制商业词书的释义体系、记忆文案、例句集合和排版。
- 真题只保留实现学习目的所必要的短句摘录，并记录年份、题型和来源；上线体验前需再次核查内容使用边界。
- 英美音频必须来自允许使用和分发的资源，或通过拥有相应授权的服务生成。

## 7. 数据模型

### 7.1 主词条

同一个单词只保存一份主词条，示例字段如下：

```json
{
  "id": "word_abandon",
  "word": "abandon",
  "importance": "core",
  "phonetics": {
    "uk": "音标",
    "us": "音标"
  },
  "audio": {
    "uk": "云存储文件标识",
    "us": "云存储文件标识"
  },
  "senses": [],
  "collocations": [],
  "morphology": [],
  "memoryTips": [],
  "synonyms": [],
  "antonyms": [],
  "confusables": [],
  "examExamples": [],
  "sources": []
}
```

### 7.2 词库清单

词库包不重复存储主词条，只记录：

- 词库 ID、名称和说明。
- 内容版本和数据格式版本。
- 单词 ID 与显示顺序。
- 包体大小、校验值和更新时间。
- 所依赖的主词条数据分片。

### 7.3 本地学习记录

本地按单词 ID 保存：

- 熟悉度状态。
- 最后标记时间。
- 最近测试结果。

本地按词库 ID 保存：

- 阅读锚点或顶部可见词条 ID。
- 字号和发音偏好。
- 已安装版本。

学习记录与词库内容分开保存。更新词库时按稳定的单词 ID 合并，不能覆盖熟悉度和阅读位置。

## 8. 技术架构

### 8.1 客户端

- 原生微信小程序。
- 页面层负责首页、词库中心、连续阅读、分类复习和测试。
- 服务层负责词库下载、版本检查、音频缓存和本地数据迁移。
- 数据层统一封装微信本地存储接口，避免页面直接读写原始键值。

### 8.2 微信云开发

- 云数据库或静态清单：保存词库元数据和当前版本清单。
- 云存储：保存分片后的词条 JSON、词库索引和音频资源。
- 云函数：首版仅用于确有必要的版本清单或临时下载地址处理，避免引入用户业务数据。

### 8.3 离线策略

- 词库下载完成后，正文和索引可完全离线使用。
- 发音按需缓存，不强制随词库全部下载。
- 无网络时允许阅读、搜索、筛选、标记和使用已有缓存音频。
- 网络恢复后由用户手动检查词库更新。

### 8.4 本地容量策略

- 词库正文按分片保存，避免单一存储项过大。
- 搜索索引采用轻量结构，避免重复保存完整词条。
- 音频设置独立缓存上限，并优先淘汰最久未使用的音频。
- 达到本地限制时提示用户删除音频缓存或不常用词库。

## 9. 异常处理

- 词库下载中断：删除临时数据，保留旧版本，并允许重试。
- 词库更新失败：继续使用旧版本，不影响学习记录。
- 数据包校验失败：拒绝安装并提示重新下载。
- 音频不可用：显示简短提示，不阻塞其他功能。
- 本地空间不足：提示预计所需空间，并提供清理入口。
- 本地记录格式升级失败：保留可读取的旧数据并停止覆盖，提示用户稍后重试。
- 微信缓存被清理或小程序被删除：本地学习记录可能永久丢失，设置页必须明确说明。
- 搜索无结果或筛选为空：提供清空条件和返回词库入口。

## 10. 跨平台与性能要求

- 兼容主流 iOS 和 Android 微信客户端。
- 适配不同安全区域、屏幕宽度、系统字号和像素密度。
- 不使用未经验证的平台专属能力。
- 连续阅读采用窗口化渲染或分页追加，控制页面节点数量。
- 搜索和熟悉度筛选在 5000–10000 条词的本地词库规模下保持可用。
- 分别验证 iOS 与 Android 的音频生命周期、本地存储限制、滚动位置恢复和后台切换行为。

## 11. 测试与验收

### 11.1 自动测试

至少覆盖：

- 词库清单解析和数据包校验。
- 单词搜索、字母索引和多条件筛选。
- 熟悉度保存与读取。
- 词库版本升级后学习记录合并。
- 测试题生成、计分和错词处理。
- 本地数据格式迁移。

### 11.2 真机验收

至少使用一台 iPhone 和一台 Android 手机验证：

- 下载、更新和删除词库。
- 断网阅读、搜索、筛选和熟悉度标记。
- 阅读位置恢复。
- 英音、美音播放和缓存。
- 长列表滚动与快速跳转。
- 前后台切换后的状态恢复。
- 两类选择题测试及错词处理。

### 11.3 首版完成标准

- 500–1000 个完整试运行词条可稳定下载和阅读。
- 每个词条按独立模块正确排版，空栏目自动隐藏。
- 已下载词库断网可用。
- 三档标记、分类复习、搜索、字母索引和阅读位置恢复可用。
- 英音、美音可播放并缓存。
- 英选中、中选英测试可完成并正确计分。
- 词库更新不覆盖用户熟悉度和阅读位置。
- iOS 与 Android 真机无阻断使用的兼容性问题。
- 所有真题短句和外部资源具有来源记录。

## 12. 推荐实施阶段

1. 项目骨架与本地数据层：建立页面结构、统一存储接口和数据格式。
2. 词库基础设施：完成云端清单、下载、校验、安装、更新和删除。
3. 词书阅读：完成词条模块、连续滚动、搜索、索引、筛选和阅读位置。
4. 学习功能：完成三档标记、分类复习和本地数据迁移。
5. 发音：接入合法英美音频资源并实现缓存管理。
6. 测试功能：完成两类选择题、计分和错词处理。
7. 内容试运行：导入 500–1000 个核心词并进行内容校验。
8. 跨平台验收：完成 iPhone 与 Android 微信真机测试和性能优化。
9. 扩充词库：在体验验证通过后补齐大纲词汇与深度内容。

## 13. 关键限制

- 本地学习记录无法在换机、清理微信缓存或删除小程序后恢复。
- 体验版用户和权限由微信小程序后台管理。
- 词库内容整理、来源核验和音频授权可能比客户端开发更耗时，应作为独立工作流管理。
- “2027 大纲词汇”名称必须与实际采用的官方大纲年份一致，不得在官方材料未发布时暗示其已发布。

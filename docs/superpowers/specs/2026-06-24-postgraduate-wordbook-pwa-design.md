# 2027 考研英语词书 PWA 设计规格

## 1. 目标

把现有微信小程序方案降级为一个自用 PWA：手机浏览器打开即可使用，iPhone 可通过 Safari“添加到主屏幕”，Android 可通过 Chrome/Edge 添加到桌面。首要目标是绕开小程序备案、AppID、云开发和体验版上传流程，让用户尽快在手机上背单词。

PWA 保留已确认的产品形态：像纸质词书一样连续阅读，不做翻面卡片；每个单词一个模块；支持三档熟悉度、分类复习、搜索、双向选择题和发音。学习数据只保存在当前浏览器本地，不提供登录、云同步、好友或商业化功能。

## 2. 使用方式与发布边界

- 使用对象：用户本人自用，必要时可分享给少数朋友。
- 运行环境：iPhone Safari、Android Chrome/Edge，以及桌面浏览器调试环境。
- 发布方式：静态网页部署。本地开发时用 `npm run web:dev`；正式自用时可放到局域网、GitHub Pages、Vercel、Netlify 或任意静态服务器。
- 安装方式：通过浏览器“添加到主屏幕”，以独立窗口方式打开。
- 数据边界：学习记录、阅读位置、设置和测试结果保存在浏览器本地 IndexedDB/localStorage。换浏览器、清缓存或换手机会丢失数据，首版不做导出导入。
- 合规边界：不复制商业词书内容；复用当前项目已审核词库和音频归因记录。

## 3. 推荐方案

采用独立 `web/` 应用，而不是把微信小程序代码硬改成网页。原因：

1. 微信小程序运行时和 Web API 差异大，直接兼容会产生大量适配层。
2. PWA 的核心收益是快速自用，独立 Web 应用更容易部署、调试和离线缓存。
3. 现有内容构建产物、词条数据、音频文件和领域逻辑可以复用，不需要重做词库。

技术选型：

- Vite + React + TypeScript，构建手机优先的单页应用。
- IndexedDB 保存词库、词条、学习状态和音频缓存索引；localStorage 保存轻量设置。
- Service Worker 负责离线缓存应用壳、词库 JSON 和常用音频。
- 复用 `content/dist` 作为静态内容来源；构建时复制到 `web/public/content`。
- 页面样式使用普通 CSS，不引入重型 UI 框架。

## 4. 首版功能范围

### 4.1 首页

首页展示：

- 已安装或可用词库。
- 继续阅读入口。
- 熟悉、待巩固、陌生三档统计。
- 词书阅读、分类复习、测试和设置入口。

如果首次打开还没有本地词库，首页显示“载入内置词库”按钮，从静态内容目录导入 800 词数据。

### 4.2 词书阅读

阅读页采用连续纵向排版，每个单词是一个模块。模块包含：

- 英文单词、音标、英音/美音按钮。
- 词性、中文释义。
- 搭配、近反义、词根词缀或记忆提示；字段为空时隐藏。
- 真题短句或例句；有来源字段时显示来源。
- 三档标记按钮：熟悉、待巩固、陌生。

页面支持英文/中文搜索、首字母筛选和熟悉度筛选。滚动位置按词库保存，重新打开后回到上次阅读附近。

### 4.3 分类复习

复习页不复制阅读实现，而是复用同一阅读页，只改变过滤条件。用户可以选择：

- 词库范围。
- 熟悉度范围：熟悉、待巩固、陌生，可多选。
- 搜索或首字母过滤。

筛选结果仍按词库原顺序阅读。

### 4.4 测试

首版提供两种选择题：

- 看英文选中文释义。
- 看中文释义选英文。

测试范围来自当前词库和熟悉度筛选。每题默认 4 个选项；候选不足时允许 2–3 个选项，不重复选项。测试结束展示总题数、正确数、正确率和错词列表，错词可批量标为待巩固或陌生。

### 4.5 发音

优先播放已有本地静态音频文件。若某个单词缺少英音或美音，只隐藏对应播放按钮，不阻断阅读。

浏览器音频策略要求用户手势触发播放，因此音频只在点击按钮后播放，不做自动播放。快速切换英音/美音时停止上一段音频。

### 4.6 离线能力

Service Worker 缓存：

- 应用壳：HTML、CSS、JS、图标、manifest。
- 当前版本词库 JSON。
- 已播放过或用户主动缓存过的音频。

离线时：

- 已导入的词库可阅读、搜索、标记和测试。
- 已缓存音频可播放。
- 未缓存音频显示非阻断提示。

## 5. 数据模型

沿用当前项目已有语义：

```ts
type Familiarity = 'familiar' | 'review' | 'unknown';

type WordState = {
  wordId: string;
  familiarity?: Familiarity;
  updatedAt: number;
};

type ProgressState = {
  libraryId: string;
  anchorWordId: string;
  scrollOffset: number;
  updatedAt: number;
};
```

IndexedDB stores：

- `libraries`：已导入词库 manifest。
- `words`：词条详情，按 `wordId` 查询。
- `libraryOrders`：词库排序 ID 列表。
- `searchIndex`：轻量搜索索引。
- `learningStates`：三档熟悉度。
- `progressStates`：阅读位置。
- `quizResults`：最近一次测试结果。
- `audioCache`：音频缓存索引。

## 6. 页面与模块边界

`web/` 目录建议结构：

```text
web/
├─ index.html
├─ package.json
├─ vite.config.ts
├─ public/
│  ├─ manifest.webmanifest
│  ├─ icons/
│  └─ content/
├─ src/
│  ├─ app/
│  ├─ components/
│  │  └─ WordEntry/
│  ├─ domain/
│  ├─ repositories/
│  ├─ services/
│  ├─ pages/
│  └─ styles/
└─ tests/
```

关键边界：

- `domain/` 只放纯函数：搜索、筛选、测试出题、字段校验。
- `repositories/` 封装 IndexedDB/localStorage。
- `services/` 组合仓储、内容导入、音频播放和离线缓存。
- `pages/` 负责页面状态和路由。
- `components/WordEntry` 只负责单词模块展示和用户操作事件。

## 7. 视觉与交互

整体风格参考纸质词书，避免卡片翻面感：

- 背景使用浅米色或暖白。
- 单词模块之间用细分割线、浅色块或章节式间距区分。
- 主色采用红橙色，呼应考研词书；操作状态使用稳定的低饱和颜色。
- 底部固定导航或首页入口保持简单，优先照顾单手浏览。
- iPhone 刘海屏和 Android 小屏通过 `safe-area-inset-*` 与响应式布局适配。

字号设置至少提供小、标准、大三档。词条内容不使用过小灰字，保证长时间阅读舒适。

## 8. 错误处理

- 内容导入失败：保留旧词库，显示可重试提示。
- IndexedDB 不可用：提示浏览器不支持本地数据库，不继续导入。
- 存储空间不足：保留学习记录，提示清理音频缓存。
- 音频缺失或播放失败：只提示当前音频不可用，不影响标记和阅读。
- Service Worker 更新：下次打开自动使用新版本；首版不做复杂更新弹窗。

## 9. 测试与验收

自动测试：

- 搜索排序和组合筛选。
- 三档标记幂等。
- 阅读位置保存和恢复。
- 双向选择题出题、干扰项和计分。
- 内容导入失败不覆盖旧版本。
- IndexedDB 仓储读写。

手动验收：

- iPhone Safari 添加到主屏幕后可打开。
- Android Chrome 添加到桌面后可打开。
- 断网后已导入词库可阅读、搜索、标记和测试。
- 已缓存音频断网可播放。
- 800 词连续滚动 3 分钟无明显卡顿。
- 小/标准/大字号均不破坏布局。

## 10. 首版不做

- 微信小程序上传、备案、体验版和云开发。
- 登录、账号、云同步、导出导入。
- 付费、分享裂变、排行榜。
- 自动复习计划、每日提醒。
- 复杂动画、卡片翻面、拼写题和听力题。
- 后台管理系统或在线编辑词库。

## 11. 完成判定

首版 PWA 只有同时满足以下条件才算可用：

- `web` 应用可本地启动并构建静态产物。
- 首次打开可导入当前 800 词内容。
- iPhone 和 Android 浏览器均可阅读、标记、搜索、复习和测试。
- 离线后已导入词库仍可用。
- 学习记录刷新页面后不丢失。
- 自动测试通过。
- README 写明本地启动、静态部署和添加到主屏幕步骤。
